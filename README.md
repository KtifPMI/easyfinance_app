# EasyFinance App

Мобильное приложение (React Native + Expo) для учёта финансов, использующее
реальный API EasyFinance.ru через собственный backend-proxy.

Структура проекта:

- `/src` — мобильное приложение (Expo)
- `/server` — backend-proxy, который безопасно подписывает запросы к EasyFinance API

---

## 1. Зачем нужен `/server`

EasyFinance API требует подпись запроса (`sig`), которая считается с
использованием секретного ключа приложения (`secret_key`). **Хранить этот
ключ внутри мобильного приложения нельзя** — его можно достать из любого
APK/IPA. Поэтому подпись считает маленький backend (`/server`), а телефон
обращается только к нему.

```
Телефон (Expo Go)  --->  /server (proxy, знает secret_key)  --->  api.easyfinance.ru
```

---

## 2. Как запустить backend (`/server`)

1. Зарегистрируйте приложение на https://easyfinance.ru/my/register-app/ —
   вы получите `app_id` и `secret_key`.
2. Откройте терминал и перейдите в папку сервера:
   ```bash
   cd server
   npm install
   ```
3. Скопируйте файл с настройками:
   ```bash
   cp .env.example .env
   ```
4. Откройте `.env` и впишите свои значения:
   ```
   EASYFINANCE_APP_ID=ваш_app_id
   EASYFINANCE_SECRET_KEY=ваш_secret_key
   ```
5. Запустите сервер:
   ```bash
   npm start
   ```
   Если всё хорошо, увидите:
   ```
   easyfinance-proxy listening on http://localhost:4000
   ```

Оставьте это окно терминала открытым — сервер должен работать, пока вы
пользуетесь приложением.

---

## 3. Как запустить мобильное приложение

1. Откройте **новое** окно терминала, перейдите в корень проекта:
   ```bash
   cd /Users/milena/EasyFinanceApp
   ```
2. Узнайте локальный IP-адрес своего компьютера (он понадобится телефону,
   чтобы достучаться до `/server`):
   - Mac: `ipconfig getifaddr en0` → например `192.168.1.13`
3. Запустите приложение, указав адрес сервера:
   ```bash
   EXPO_PUBLIC_PROXY_URL=http://192.168.1.13:4000 npx expo start
   ```
   (замените IP на свой; компьютер и телефон должны быть в одной Wi-Fi сети)

---

## 4. Как открыть приложение через Expo Go

1. Установите приложение **Expo Go** из App Store / Google Play.
2. После `npx expo start` в терминале появится QR-код.
3. Откройте Expo Go → отсканируйте QR-код (на iOS — через стандартную камеру).
4. Приложение загрузится на телефон.

---

## 5. Вход в приложение (авторизация EasyFinance)

При первом запуске откроется экран "Войти через EasyFinance":

1. Нажмите **"Войти через EasyFinance"**.
2. Откроется браузер со страницей easyfinance.ru — войдите под своим
   логином/паролем easyfinance.ru и разрешите доступ приложению.
3. После подтверждения браузер должен вернуться в приложение автоматически.
   Если этого не произошло — на экране результата скопируйте значение
   параметра `code` из адресной строки и вставьте его в поле "Код
   подтверждения" в приложении.

---

## 6. Куда вставлять `app_id` и `secret_key`

Только в `/server/.env` (см. шаг 4 раздела 2). В коде мобильного
приложения этих значений нет и быть не должно.

## 7. Статус проекта

### ✅ Что уже работает
- Backend-proxy: подпись запросов (sig), OAuth-обмен code → access_token,
  универсальный прокси `/api/call` к любому методу EasyFinance API.
- Авторизация через системный браузер (без WebView), сохранение
  `access_token`/`uid` в SecureStore/AsyncStorage.
- Главный экран, профиль пользователя (`users.get`).
- Счета (`accounts.get`).
- Операции: список, добавление, редактирование, удаление, корзина
  (`operations.get/post/set`).
- Категории (`categories.get`).
- Базовый бюджет (`budget.get`, с fallback на локальные данные, если API
  не возвращает категории бюджета).
- Цели, календарь событий, рекомендации, курсы валют, EasyBank — работают
  на встроенных мок-данных (этих сущностей нет в EasyFinance API).

### ⚠️ Требует проверки на реальном аккаунте
- Точная формула `sig` реализована по примеру из документации
  (`md5(secret_key + uid + params)`) — нужно проверить на реальных
  `app_id`/`secret_key`.
- OAuth-редирект: неизвестно, поддерживает ли EasyFinance кастомный
  `redirect_uri` на схему приложения (`easyfinanceapp://oauth-callback`).
  Если нет — используйте ручной ввод `code` (см. раздел 5).
- Маппинг полей категорий (`type`, `icon`, цвет) и счетов (иконки/цвета по
  `type_id`) — сделаны по неполным данным документации, могут потребовать
  корректировки в `src/services/api/finance.ts`.
- Создание/редактирование операций типа "перевод" (`transfer`) — логика
  реализована по описанию `transfer_account_id`/`transfer_amount`, но не
  протестирована вживую.
